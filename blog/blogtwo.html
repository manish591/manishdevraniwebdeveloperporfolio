All You Need To Know About CSS Selectors

Written By Manish Devrani

For those of you who don't know, CSS is a style sheet language. It allows you to transfer your ideas into a web format. CSS allows you to add colors, fonts, borders etc to your web page i.e. style your web page. CSS has been around for a very long time and has gone through major changes since it's first release. This article will discuss about css selector, one of the core concept of learning css, so let's begin.

The first and the biggest key to understanding CSS is understanding selectors. Selectors are what allows you to target specific HTML elements and apply style to them. This blog is for absolute beginners, You will surely learn about CSS selectors after that.

There are several type of selector in CSS.

  1. 1. ID Selector
  2. 2. Tag Selector
  3. 3. Class Selector
  4. 4. Universal selector
  5. 5. Attribute Selector

ID Selector

Syntax

#id_value { style properties }

You can see below how we are selcting elements with id selector.

The CSS ID selector matches an element based on the value of the element’s id attribute. In order for the element to be selected, its id attribute must match exactly the value given in the selector. ID selectors are the most powerful type of selector in terms of CSS specificity. Meaning that they beat out other types of selectors and the styles defined within win. That sounds good, but that’s typically considered bad, because it’s nice to have lower-specificity selectors that are easier to override when needed.

Tag Selector

Syntax

element { style properties }

You can see below how we are selcting elements with type selector.

The CSS type selector matches elements by node name. In other words, it selects all elements of the given type within a document. Tag selectors are at their most useful when changing properties that are unique to that HTML element. Like setting the list-style on a or tab-size on a . Also in reset stylesheets where you are specifically trying to unset styles that browsers apply to certain elements. Don’t rely on them too much though. It’s typically more useful to have a class define styling that you can use on any HTML element.

Class Selector

Syntax

.class_name { style properties }

You can see below how we are selcting elements with id selector.

The CSS class selector matches elements based on the contents of their class attribute. Class selectors are your friend. They are probably the most useful and versatile selectors out there. In part because they are well supported in all browsers. In part because you can add multiple classes (just separated by a space) on HTML elements. In part because there are JavaScript things you can do specifically for manipulating classes.

Universal Selector

Syntax

* { style properties }

You can see below how we are selcting elements with id selector.

The CSS universal selector (*) matches elements of any type. Universal selectors can be namespaced when using @namespace. This is useful when dealing with documents containing multiple namespaces such as HTML5 with inline SVG or MathML, or XML that mixes multiple vocabularies.

Attribute Selector

Syntax

[attr=value] { style properties }

You can see below how we are selcting elements with id selector.

The CSS attribute selector matches elements based on the presence or value of a given attribute. You might argue that attribute selectors are even more useful than classes because they have the same specificity value, but can be any attribute not just class, plus they can have a value you can select by.
